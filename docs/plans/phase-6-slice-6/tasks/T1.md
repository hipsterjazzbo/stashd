# T1 · Auto-download eligible items at end of input commit

**Bug:** `CreateStashFromDiscovery::commit()` creates items but never enqueues `item.download`.
`DownloadPolicyEvaluator::allowsAutomaticDownload()` exists, is correct, only called in tests.

**Do** — at the **end** of the input-commit (after `stash_items` exist), in `CreateStashFromDiscovery`:
- Collect `mediaItemId`s of newly-created stash items (the `findByStashAndMediaItem(...) === null`
  branch) that are not filtered out (`state !== ignored`).
- If `DownloadPolicyEvaluator::allowsAutomaticDownload($stash->downloadPolicy)`: inject
  `CommandDispatchService`, enqueue one download each via
  `dispatch(CommandType::ItemDownload, ['mediaItemId' => …, 'stashId' => …])`. Reuse this path only
  (runs `ItemDownloadCommandHandler::validate()` + `createJobs()`). No parallel download path; respect
  serial worker + idempotency.
- `metadata_only` / `manual_download` → enqueue nothing.

**Files:** `app/Stashes/CreateStashFromDiscovery.php`, `app/Downloads/DownloadPolicyEvaluator.php`,
`app/Downloads/ItemDownloadCommandHandler.php`, `app/Commands/CommandDispatchService.php`.

**Done when:** fake-provider stash + `video` policy → downloads run automatically, Vault assets reach
`ready`, no manual command. Empty / metadata-only → nothing enqueued.
