# T3 · Slug ordinal-suffix fallback (filesystem-style, starts at 1)

Replace the duplicate-slug throw with a helper yielding `foo`, then `foo-1`, `foo-2`, … (ordinal
**starts at 1**). After the reframe the only slug-creation site is `POST /stashes` (T8) — centralize
there. Remove the now-dead throw in `CreateStashFromDiscovery`.

**Implementation note — prefer SQL over an N-roundtrip loop:** add
`StashRepository::nextAvailableSlug(string $base)` using one query
(`WHERE slug = ? OR slug LIKE 'base-%'`) and compute the next free ordinal in PHP. Do **not** use a
naive `COUNT` (wrong with gaps). Handle edge cases: gaps from deletions, and literal `-N` user slugs
colliding with the generated pattern.

**Files:** `app/Stashes/StashRepository.php`, `StashController.php` (T8 call site),
`CreateStashFromDiscovery.php`.

**Done when:** two stashes slugifying to `foo` → `foo`, `foo-1`; a third → `foo-2`; no error.
