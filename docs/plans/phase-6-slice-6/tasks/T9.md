# T9 · Add-input pipeline (retire create_from_preflight; commit into existing stash)

The keystone. Splits atomic create into `POST /stashes` (T8, empty) + `POST /stashes/{id}/inputs`
(add input). Preflight stays stash-agnostic; commit targets a stash id.

**Do:**
- **Repurpose** create-from-preflight machinery into add-input:
  `StashCreateFromPreflightCommandHandler` → `StashAddInputCommandHandler` (validate stash exists +
  preflight completed); `CreateFromPreflightJobHandler` → `AddInputJobHandler`; rename
  `CommandType::StashCreateFromPreflight` → `StashAddInput`, `JobIntent::CreateFromPreflight` →
  `AddInput`. Update `CommandHandlerRegistryInitializer` + `JobHandlerRegistryInitializer`.
- Refactor `CreateStashFromDiscovery`: replace `commit()` with
  `commitInput(StashRecord $stash, PrefixedUlid $preflightCommandId, array $options)` — load resolved
  identity from preflight, **re-run full discovery** (best strategy, T5), apply filters (T7), create
  `stash_input` (with options) + media items + sources + stash items into the **existing** stash, set
  stash `icon_uri` from avatar if unset, then enqueue downloads (T1) using `stash->downloadPolicy`.
- **Provider `preview` capability (provider-agnostic):** add a generic preview method to the
  `Provider` interface → identity (source title, avatar, estimated count) + small sample, implemented
  per provider (YouTube via T2 page parse; Fake via `resolveInput` + bounded `discover`). Preflight
  calls this — it must not know about YouTube. **Unsupported URL:** `ProviderRegistry::resolveForUri`
  throws today — catch on the preflight path, return stable `provider_unsupported` error so the
  frontend shows a "not supported yet — request it" affordance.
- `POST /api/v1/stashes/{id}/inputs {preflight_command_id, options}` → dispatch `stash.add_input`.
  Detail page renders the review card + always-available "+ Add input" (with "No inputs configured"
  empty state).
- Data model already supports many inputs + dedup (`findByProviderIdentity`,
  `findByStashAndMediaItem`) — preserved. **Keep `stash_input` ids stable/referenceable** so a
  broadcast can map input → season (T20).

**Depends on:** T8, T2, T5. **Pairs with:** T7.

**Done when:** a stash holds ≥2 inputs from different sources; items attributed to the right input;
dedup across inputs preserved; auto-download fires per committed input; unsupported URL →
`provider_unsupported` + request-support UI.
