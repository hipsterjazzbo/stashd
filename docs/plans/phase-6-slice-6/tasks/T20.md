# T20 · Optional input → broadcast season mapping (deferrable within this slice)

A stash's multiple inputs can optionally map to **seasons** in series-style media-server broadcasts
(`FilesystemSeries`/`JellyfinSeries`/`PlexSeries`). This is **broadcast config**, stored in the
existing `BroadcastRecord.settingsJson` (no schema change) as `stash_input_id → season`.

**Do:**
- Persist the mapping in `BroadcastRecord.settingsJson`.
- Generation (`BroadcastContextFactory` / `BroadcastPathBuilder`) honours the mapping when present;
  else falls back to current behaviour.
- UI: per-input season assignment in the broadcast config on `stash-detail.view.php`.

**Scope note:** largest net-new behaviour in the slice — **safe to defer** to a follow-up if scope
balloons. But T9 must preserve stable `stash_input` ids regardless, so the mapping has something to
reference. If deferred, ship the stable-id guarantee now.

**Depends on:** T9 (stable input ids).

**Done when:** a series broadcast can assign each input to a season; generated layout reflects the
mapping; unmapped inputs fall back to default organization.
