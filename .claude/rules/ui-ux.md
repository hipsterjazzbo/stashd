# UI / UX rules

Stashd should feel calm, dense, accessible, and trustworthy.

Brand direction:

```text
stashd_
Because the internet forgets.
```

Visual feel:

- warm dark/amber
- understated self-hosted polish
- Glance-inspired warmth
- not cold enterprise SaaS
- not YouTube-branded

UX priorities:

- show progress clearly for long-running work
- explain storage impact before expensive choices
- surface feed URLs so users can recover them
- distinguish unavailable storage roots from individual missing assets
- prefer clear recovery actions over scary errors
- use domain language consistently: Stash, Vault, Broadcast

Avoid:

- loud marketing copy
- vague enterprise dashboards
- destructive or public-by-default flows
- hidden duplication of large media files
